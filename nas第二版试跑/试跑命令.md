# NAS 第二版试跑命令

本文档用于记录第二版 NAS 试跑时最常用的一组命令。

第二版的核心变化：

- 搜索阶段支持 `--exhaustive_search`，可对 2880 个离散候选全部做 proxy 评估。
- 结果单独保存到 `runs/nas_search_v2/` 下，不和第一版 `runs/nas_search/` 混在一起。
- 后续重训练和 benchmark 仍沿用现有脚本，只是输入路径改成第二版搜索结果目录。

## 1. 推荐主命令：全量 proxy 搜索

```bash
python scripts/nas/search_channel_aware.py \
  --dataset_name UCMerced_LandUse \
  --train_dir data/UCMerced_LandUse/UCMerced_LandUse-train \
  --valid_dir data/UCMerced_LandUse/UCMerced_LandUse-valid \
  --output_dir runs/nas_search_v2 \
  --device cuda:0 \
  --image_size 256 \
  --batch_size 80 \
  --num_workers 6 \
  --exhaustive_search \
  --search_epochs 3 \
  --top_k 8 \
  --lambda_param 0.01 \
  --lambda_tx 0.01 \
  --lambda_robust 0.2 \
  --lambda_cr 0.2 \
  --target_cr 0.7 \
  --min_dynamic_cr 0.3 \
  --max_dynamic_cr 1.0 \
  --rate_blend_alpha 0.7
```

结果目录示例：

```text
runs/nas_search_v2/exhaustive_search/UCMerced_LandUse_YYYYMMDD_HHMMSS/
```

搜索结束后，重点关注这些文件：

- `search_results.jsonl`：全部候选的原始结果。
- `ranked_results.json` / `ranked_results.csv`：按 score 排序后的完整排名。
- `best_arch.json`：当前加权 score 最优的架构。
- `topk_arches.json`：Top-k 架构列表。
- `summary.json`：运行汇总。

## 2. 快速试跑命令：先检查流程是否正常

如果只是想先确认脚本、显存、数据路径都正常，可以先跑一个轻量版本：

```bash
python scripts/nas/search_channel_aware.py \
  --dataset_name UCMerced_LandUse \
  --train_dir data/UCMerced_LandUse/UCMerced_LandUse-train \
  --valid_dir data/UCMerced_LandUse/UCMerced_LandUse-valid \
  --output_dir runs/nas_search_v2 \
  --device cuda:0 \
  --image_size 224 \
  --batch_size 32 \
  --num_workers 4 \
  --exhaustive_search \
  --search_epochs 1 \
  --max_train_batches 40 \
  --max_eval_batches 12 \
  --top_k 5
```

这个命令的目标不是拿最终结果，而是：

- 看是否会 OOM
- 看单候选耗时大概多少
- 看日志和结果文件是否正常生成

## 3. Pareto 分析

假设第二版搜索目录是：

```text
runs/nas_search_v2/exhaustive_search/UCMerced_LandUse_YYYYMMDD_HHMMSS/
```

可以继续做 Pareto 筛选：

```bash
python scripts/nas/evaluate_pareto.py \
  --results_jsonl runs/nas_search_v2/exhaustive_search/UCMerced_LandUse_YYYYMMDD_HHMMSS/search_results.jsonl \
  --top_k 20
```

会生成：

- `pareto_front.json`
- `pareto_topk.json`
- `pareto_summary.md`

## 4. 重训练 Top-1

先对 `best_arch.json` 做完整重训练：

```bash
python scripts/nas/retrain_candidate.py \
  --arch_json runs/nas_search_v2/exhaustive_search/UCMerced_LandUse_YYYYMMDD_HHMMSS/best_arch.json \
  --dataset_name UCMerced_LandUse \
  --train_dir data/UCMerced_LandUse/UCMerced_LandUse-train \
  --valid_dir data/UCMerced_LandUse/UCMerced_LandUse-valid \
  --output_dir runs/nas_retrain_v2 \
  --device cuda:0 \
  --epochs 120 \
  --batch_size 64 \
  --num_workers 6

如果想先做快速冒烟测试：

```

## 5. 批量重训练 Top-3

下面这组命令会从 `ranked_results.json` 提取 Top-3，并逐个重训练：

```bash
RUN_DIR="runs/nas_search_v2/exhaustive_search/UCMerced_LandUse_YYYYMMDD_HHMMSS"
OUT_ROOT="runs/nas_retrain_v2_top3"
mkdir -p "${OUT_ROOT}/arch_jsons"

python - <<'PY'
import json
from pathlib import Path

run_dir = Path("runs/nas_search_v2/exhaustive_search/UCMerced_LandUse_YYYYMMDD_HHMMSS")
rows = json.loads((run_dir / "ranked_results.json").read_text(encoding="utf-8"))
out = Path("runs/nas_retrain_v2_top3/arch_jsons")
out.mkdir(parents=True, exist_ok=True)
for i, row in enumerate(rows[:3], start=1):
    (out / f"top{i}_{row['arch_tag']}.json").write_text(
        json.dumps(row["arch"], indent=2), encoding="utf-8"
    )
PY

for arch in ${OUT_ROOT}/arch_jsons/top*.json; do
  python scripts/nas/retrain_candidate.py \
    --arch_json "$arch" \
    --dataset_name UCMerced_LandUse \
    --train_dir data/UCMerced_LandUse/UCMerced_LandUse-train \
    --valid_dir data/UCMerced_LandUse/UCMerced_LandUse-valid \
    --output_dir "${OUT_ROOT}" \
    --device cuda:0 \
    --epochs 120 \
    --batch_size 64 \
    --num_workers 6
done
```

## 6. Benchmark 命令

### 6.1 Legacy vs 第二版 NAS

```bash
python scripts/benchmark/compare_models.py \
  --data_dir data/UCMerced_LandUse/UCMerced_LandUse-valid \
  --output_dir runs/model_benchmark_v2 \
  --run_name legacy_vs_nas_v2_best \
  --device cuda:0 \
  --batch_size 64 \
  --num_workers 6 \
  --image_size 256 \
  --mc_runs 5 \
  --channel_types AWGN,Fading,Combined_channel \
  --model_a_name legacy_baseline \
  --model_a_type legacy \
  --model_a_ckpt checkpoint/your_legacy_best.pth \
  --model_a_legacy_cr 0.8 \
  --model_b_name nas_v2_best \
  --model_b_type nas \
  --model_b_ckpt runs/nas_retrain_v2/<run_id>/best_model.pth \
  --model_b_arch_json runs/nas_search_v2/exhaustive_search/UCMerced_LandUse_YYYYMMDD_HHMMSS/best_arch.json
```

### 6.2 第二版 Top-1 vs Top-2

```bash
python scripts/benchmark/compare_models.py \
  --data_dir data/UCMerced_LandUse/UCMerced_LandUse-valid \
  --output_dir runs/model_benchmark_v2 \
  --run_name nas_v2_top1_vs_top2 \
  --device cuda:0 \
  --mc_runs 5 \
  --model_a_name nas_v2_top1 \
  --model_a_type nas \
  --model_a_ckpt runs/nas_retrain_v2/<top1_run>/best_model.pth \
  --model_a_arch_json runs/nas_search_v2/exhaustive_search/UCMerced_LandUse_YYYYMMDD_HHMMSS/best_arch.json \
  --model_b_name nas_v2_top2 \
  --model_b_type nas \
  --model_b_ckpt runs/nas_retrain_v2/<top2_run>/best_model.pth \
  --model_b_arch_json runs/nas_retrain_v2_top3/arch_jsons/top2_<tag>.json
```

## 8. 结果目录建议

为了避免第二版和第一版混淆，建议统一按下面方式保存：

```text
runs/nas_search_v2/exhaustive_search/<search_run_id>/
runs/nas_retrain_v2/<retrain_run_id>/
runs/nas_retrain_v2_top3/
runs/model_benchmark_v2/<benchmark_run_id>/
```

## 9. 试跑建议

- 第一次开跑前，先用“快速试跑命令”确认环境和显存都正常。
- 全量搜索完成后，优先查看 `ranked_results.csv`、`topk_arches.json` 和 `pareto_front.json`。
- 真正写结论时，不要只看 proxy 排名，最好至少重训练 Top-3。
- benchmark 时保持 `data_dir`、`batch_size`、`image_size` 一致，避免比较不公平。
