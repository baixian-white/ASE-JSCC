# ASE-JSCC 项目说明书（重构后）

## 1. 项目简介

- 项目名称：`ASE-JSCC`
- 目标：在遥感图像分类任务中，模拟“语义特征压缩 + 信道传输 + 分类恢复”流程。
- 核心思路：`ResNet18特征 -> SE注意力通道筛选 -> Autoencoder压缩/解压 -> 信道扰动 -> 分类`。

---

## 2. 代码结构与每个代码文件作用

### 2.1 训练脚本

- `scripts/train/ASE-JSCCtrain.py`
  - 项目主训练入口。
  - 包含：信道模块（AWGN/Fading/Combined）、SE模块、自编码器、完整分类模型。
  - 支持：从零训练（`--task train`）与断点续训（`--task continue`）。
  - 输出：实验目录下 `checkpoint/*.pth`、`logs/ResNet18/*.txt`、TensorBoard 事件文件。

### 2.2 评估与可视化脚本

- `scripts/eval/confusion_tsne_UM.py`
  - 在 UCMerced 测试集上生成混淆矩阵与 t-SNE 图。

- `scripts/eval/confusion_tsne_AID.py`
  - 在 AID 测试集上生成混淆矩阵与 t-SNE 图。

- `scripts/eval/confusion_tsne_Soya.py`
  - 在 SoyaHealthVision 测试集上生成混淆矩阵与 t-SNE 图。

### 2.3 可解释性脚本

- `scripts/visualization/grad_cam_visualize.py`
  - 进行多层 Grad-CAM 可视化（默认 `layer2/layer3/layer4`）。
  - 支持指定目标类别、指定样本索引或随机抽样。

### 2.4 推理脚本

- `scripts/infer/predict_demo.py`
  - 支持单图预测与 ImageFolder 批量预测。
  - 支持导出 CSV 结果。

### 2.5 数据划分脚本

- `scripts/data_prep/split_ucmerced_landuse.py`
  - 将 UCMerced 官方结构数据按 `5:1:4` 划分 train/valid/test。

- `scripts/data_prep/split_AID.py`
  - 将 AID 数据按 `5:1:4` 划分 train/valid/test。

- `scripts/data_prep/split_Soya.py`
  - 将 Soya 数据按 `6:3:1` 划分 train/valid/test。

### 2.6 配置、文档、Notebook

- `configs/environment.yml`：历史环境（Python3.6 + torch1.7.1）。
- `configs/ase-jscc-gpu.yml`：推荐 GPU 环境（较新 PyTorch/CUDA）。
- `docs/dataset.txt`：数据集下载链接。
- `docs/命令行提示.md`：环境、代理、TensorBoard 等命令备忘。
- `notebooks/Untitled.ipynb`：数据加载与可视化实验草稿。
- `notebooks/Untitled1.ipynb`：CUDA/显卡环境检查草稿。

---

## 3. 目录职责

- `scripts/`：所有可执行 Python 脚本。
- `configs/`：环境配置。
- `docs/`：说明文档。
- `notebooks/`：探索性 Notebook。
- `data/`：原始数据与划分数据。
- `checkpoint/`：通用模型权重。
- `logs/`：评估图与可视化产物。
- `runs/`：TensorBoard 日志。
- `<dataset>_<epoch>_<channel>_<cr>/`：分实验输出目录（含 checkpoint/logs）。

## 4. 路径策略（本次已改造）

核心脚本统一采用 `PROJECT_ROOT` 解析路径：

- 自动查找 `.git` 所在目录作为项目根。
- 数据、权重、日志路径均从项目根拼接。
- 不再依赖“当前终端所在目录”。

---

## 5. 标准使用流程

1. 环境准备：`conda env create -f configs/ase-jscc-gpu.yml`
2. 数据准备：将数据放入 `data/`，必要时运行 `scripts/data_prep/*.py`
3. 训练：运行 `scripts/train/ASE-JSCCtrain.py`
4. 评估：运行 `scripts/eval/confusion_tsne_*.py`
5. 可解释性：运行 `scripts/visualization/grad_cam_visualize.py`
6. 推理：运行 `scripts/infer/predict_demo.py`

---

## 6. 典型命令

```bash
# 训练
python scripts/train/ASE-JSCCtrain.py --task train --cr 0.8 --num_epochs 150 --channel_type Combined_channel

# 评估
python scripts/eval/confusion_tsne_UM.py
python scripts/eval/confusion_tsne_AID.py
python scripts/eval/confusion_tsne_Soya.py

# Grad-CAM
python scripts/visualization/grad_cam_visualize.py -c overpass -n 6

# 推理
python scripts/infer/predict_demo.py --checkpoint checkpoint/xxx.pth --image data/xxx.jpg
```
