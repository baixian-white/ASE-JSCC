# ASE-JSCC 项目结构说明书（重构版）

## 1. 重构目标

本次重构针对两个问题：

1. 根目录文件过多，脚本/配置/文档混放，维护成本高。
2. 多个脚本使用相对路径，执行目录变化时容易报错（找不到数据/权重/日志目录）。

重构后实现：

- 统一目录分层：`scripts / configs / docs / notebooks`
- 脚本统一使用“项目根目录路径解析”（通过 `.git` 自动定位）
- 数据与历史实验结果目录保持原位，不破坏既有产物

## 4. 代码路径改造规则

所有核心脚本已统一加入如下路径策略：

1. 自动寻找项目根目录（遍历父目录直到找到 `.git`）
2. 关键路径统一由 `PROJECT_ROOT / "..."` 构造
3. 推理脚本对命令行相对路径参数自动补齐为 `PROJECT_ROOT` 下路径

示例（逻辑一致）：

```python
PROJECT_ROOT = get_project_root()
weight_path = PROJECT_ROOT / "checkpoint" / "xxx.pth"
data_root = PROJECT_ROOT / "data" / "UCMerced_LandUse-test"
out_dir = PROJECT_ROOT / "logs" / "ResNet18"
```

这样执行时不再依赖“当前 shell 所在目录”。

## 5. 运行命令（重构后）

### 5.1 训练

```bash
python scripts/train/ASE-JSCCtrain.py --task train --cr 0.8 --num_epochs 150 --channel_type Combined_channel
```

### 5.2 评估

```bash
python scripts/eval/confusion_tsne_UM.py
python scripts/eval/confusion_tsne_AID.py
python scripts/eval/confusion_tsne_Soya.py
```

### 5.3 可解释性

```bash
python scripts/visualization/grad_cam_visualize.py -c overpass -n 6
```

### 5.4 推理

```bash
python scripts/infer/predict_demo.py --checkpoint checkpoint/xxx.pth --image data/xxx.jpg
python scripts/infer/predict_demo.py --checkpoint checkpoint/xxx.pth --data_dir data/UCMerced_LandUse-test --csv_out logs/predictions.csv
```

### 5.5 数据划分

```bash
python scripts/data_prep/split_ucmerced_landuse.py
python scripts/data_prep/split_AID.py
python scripts/data_prep/split_Soya.py data/SoyaHealthVision data/SoyaHealthVision
```

## 6. 保持不变的部分

- `data/` 数据目录（包含原始和已划分数据）
- `checkpoint/`、`logs/`、`runs/` 既有训练与评估产物
- 各历史实验目录（如 `AID_150_combine_0.8`）

本次重构未移动这些大目录，避免造成长时间 IO 和结果丢失风险。

## 7. 后续建议

1. 如需进一步工程化，可将重复模型定义抽到 `src/ase_jscc/model.py`。
