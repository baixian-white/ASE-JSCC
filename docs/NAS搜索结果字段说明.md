# NAS 搜索结果字段说明

这份表用于解释 `ranked_results.csv`、`ranked_results.json` 和分析图里常见字段的含义。

## 字段说明表

| 字段 | 含义 | 怎么看 |
| --- | --- | --- |
| `rank` | 按综合评分 `score` 排序后的名次 | 数值越小越靠前，`1` 表示当前最优候选 |
| `arch_tag` | 架构标签 | 用一串缩写编码这个候选的结构参数，便于快速识别 |
| `score` | 综合评分 | 搜索阶段的最终排序依据，综合考虑精度、参数量、传输代价和鲁棒性 |
| `mean_acc` | 平均准确率 | 在多个信道类型和多个 SNR 条件下的平均分类准确率，越大越好 |
| `worst_acc` | 最差准确率 | 所有评估条件里的最低准确率，用来衡量鲁棒性，越大越好 |
| `mean_cr` | 平均压缩率 | 动态/样本级压缩率的统计均值，反映平均保留了多少通道 |
| `std_cr` | 压缩率标准差 | 压缩率在不同样本间的波动程度，越小表示越稳定 |
| `param_m` | 参数量（百万） | 模型大小，单位是 million parameters，越小越轻量 |
| `tx_cost` | 传输代价 | 信道传输开销指标，越小越好，表示更省传输资源 |
| `robust_gap` | 鲁棒性差距 | 不同信道/SNR 条件下的性能波动，越小越稳定 |
| `total_time_sec` | 总耗时（秒） | 该候选从评估到输出结果的总运行时间，主要看效率，不看优劣 |

## `arch_tag` 的拆解方式

以 `s4_r16_cr0.4_cb24_d3_k1_skip` 为例：

| 片段 | 含义 |
| --- | --- |
| `s4` | `insertion_stage=4`，表示 NAS 模块插入到 ResNet 的第 4 个阶段 |
| `r16` | `se_ratio=16`，表示 SE/注意力模块的压缩比 |
| `cr0.4` | `cr=0.4`，表示基础压缩率设置 |
| `cb24` | `bottleneck_channels=24`，表示自编码器瓶颈通道数 |
| `d3` | `ae_depth=3`，表示自编码器深度 |
| `k1` | `kernel_size=1`，表示卷积核大小 |
| `skip` | `use_skip=True`，表示使用残差/跳连结构 |

## 汇报时怎么理解这张表

- 如果你想强调“综合最优”，看 `score`
- 如果你想强调“纯分类性能”，看 `mean_acc`
- 如果你想强调“最坏情况下也稳不稳”，看 `worst_acc` 和 `robust_gap`
- 如果你想强调“模型轻不轻”，看 `param_m`
- 如果你想强调“传输开销大不大”，看 `tx_cost`
